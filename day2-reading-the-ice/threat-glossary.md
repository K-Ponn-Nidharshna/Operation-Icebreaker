# Threat Glossary — Reading the Ice
**Operation IceBreaker | Day 2 | K. Ponn Nidharshna**
 
## CIA Triad

1. **Confidentiality** – Ensuring that information is only accessible 
   to those who are authorized to see it. Example: encrypting a 
   password file so only admins can read it.

2. **Integrity** – Making sure data is accurate and has not been 
   tampered with. Example: using file hashes to detect if a file 
   was modified.

3. **Availability** – Ensuring systems and data are accessible when 
   needed. Example: keeping a server running so users can always 
   log in.

## Authentication vs Authorization

4. **Authentication** – Verifying who you are. Example: entering 
   your username and password to prove your identity.

5. **Authorization** – Determining what you are allowed to do after 
   authentication. Example: a logged-in user can view files but 
   cannot delete them.

## Common Attacks

6. **Phishing** – A social engineering attack where an attacker sends 
   a fake email pretending to be a trusted source to steal 
   credentials or install malware.

7. **Malware** – Malicious software designed to damage, disrupt, or 
   gain unauthorized access to a system. Includes viruses, trojans, 
   and ransomware.

8. **Brute Force Attack** – An attack where the attacker tries every 
   possible password combination until the correct one is found.

9. **Man-in-the-Middle (MitM)** – An attack where the attacker 
   secretly intercepts communication between two parties without 
   either knowing.

10. **SQL Injection** – Inserting malicious SQL code into an input 
    field to manipulate a database and extract or destroy data.

11. **Cross-Site Scripting (XSS)** – Injecting malicious scripts into 
    a trusted website that then runs in the victim's browser.

12. **Denial of Service (DoS)** – Flooding a system with traffic to 
    overwhelm it and make it unavailable to legitimate users.

13. **Ransomware** – A type of malware that encrypts the victim's 
    files and demands payment to restore access.

14. **Social Engineering** – Manipulating people psychologically to 
    trick them into revealing confidential information or performing 
    actions that compromise security.

## Other Key Terms

15. **Vulnerability** – A weakness in a system that can be exploited 
    by an attacker to gain unauthorized access.

16. **Exploit** – A piece of code or technique that takes advantage 
    of a vulnerability to cause unintended behavior in a system.

17. **Firewall** – A network security tool that monitors and controls 
    incoming and outgoing traffic based on predefined security rules.

18. **Encryption** – The process of converting readable data into an 
    unreadable format using an algorithm, so only authorized parties 
    can decode it.

19. **Hashing** – A one-way process that converts data into a 
    fixed-length string. Unlike encryption, it cannot be reversed. 
    Used to store passwords securely.

20. **Zero-Day Vulnerability** – A software flaw that is unknown to 
    the vendor and has no patch available, making it extremely 
    dangerous when exploited.
